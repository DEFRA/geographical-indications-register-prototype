Version 2 design history

Overview

Before the start of this sprint, our intention was to iterate on version 1 and check if the changes worked for users. However, at the snd of the first Alpha sprint we found out about GOV Specialist Publisher and decided we should test its interface as it would appear for protected food names.

We produced an iteration of version 1 screens (custom build version) so we had something to test later if Specialst Publisher did not work for our users. We also produced a version using Specialist Publisher default view and also a set of screens with some changes to the default screens.

Changes from v1 to v2 for the custom-build screens

  • Guidance page now uses the signed-off text from the October 2019 work gthat was almost released.
  • Search uses the horizontal layout with filters initially hidden
  • Search results are shown in a table
  • We removed the 'register' column as this was not useful to users
  • Added 'Date of original registration with the EU' because users really valued this
  • Changed the product detail display to use the fields that the business told us wouldbe part of the register
  • Removed link to 'legal registers' as this tested badly - users did not understand what it meant. We linked to the registers from the guidance page.
  • Changed 'status' text to check if it would be clearer to users.
  • Opened specification document in same tabe, not a new tab.

Main research objectives

  • Can users use GOV.UK Specialist Publisher default view to find what they need?
  • Is the custom-designed Specialist Publisher view more useful to users?
  • (If there is time) Can users use the version 2 iteration of the service to complete their task? Is it an improvement over version 1?

Version 1 design history

Overview

From research in discovery, we believed that most users wanted to find the details and specification for a single product. Given that these users knew the product name (even if they couldn't always spell it) we decided to offer keyword search as the main route to the content.

We decided to start the sessions by showing people a minimum version of the search results interface (Search A with list), so we could discover what they needed. They were also shown versions with more slightly more data displayed (Search A with list).

Main research objectives

  • Can users find product information easily using the service?
  • Do users need search results with more detail than just product name?
  • Which search layout will users find most useful?
  • If they use filters, can they use them to get the information they need?

Project background design history

Background to the project

Currently the EU publishes a set of registers of geographical indications recognised within the EU. These are protected names for food and drink based on the area the product was produced in and/or the process involved in making the product.

When the transition period for leaving the EU ends the UK government is required by law to produce and publish registers of geographical indications that are recognised under the new domestic GI schemes being established.

The project to produce these registers has been with Defra Digital for several years and has been approached in a number of ways. Previous options considered included:

  • publishing as Excel / CSV documents on DATA.GOV.UK
  • storing the registers in the Reference Data Service
  • publishing them using Defra’s Data Services Platform
  • publishing the registers as GOV.UK Registers.
